# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅        |

## Reporting a Vulnerability

If you discover a security vulnerability in Mantis, please report it responsibly.

**DO NOT** open a public issue for security vulnerabilities.

### How to Report

1. **Email:** Send details to the maintainer via GitHub private vulnerability reporting
2. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 7 days
- **Fix/Patch:** Within 30 days for critical issues

## Scope

This policy applies to the Mantis CLI tool and its dependencies. It does **not** cover:

- Vulnerabilities in target applications being scanned
- Third-party services or APIs
- Social engineering attacks

## Responsible Use

Mantis is a security testing tool. Always ensure you have **explicit authorization** before scanning any application. Unauthorized security testing is illegal and unethical.
